this总结

1. this

this代表函数运行时，自动生成的一个内部对象，只能在函数内部使用。this的值可以根据函数调用的上下文而改变，即this的绑定是延迟绑定，是在函数被调用的过程中确定绑定对象的。

2. 不同情况下的this

2.1 直接调用

直接调用函数，this在严格模式下为undefined,非严格模式指向window。

tip：无论闭包中返回函数或者在函数中调用函数，无论多复杂，跟作用域链没有关系，只要函数的调用方式是直接调用，那么this的指向在非严格模式下是执行window对象，在严格模式下是指向undefined.

2.2 对象的属性

函数是对象的属性，通过对象调用函数，那么this指向的是这个对象。

tip：
当把对象的函数属性赋给变量时，变量指向这个函数，通过变量调用即为直接调用函数。

函数作为参数传入到另一个函数中，赋值给形参，形参和对象属性指向的是同一个函数。在函数中调用函数时是直接调用。

tip:当对象的函数属性赋给连另一个对象，通过另一个对象调用这个函数时，this指向的就是调用这个函数的对象。

2.3 构造函数

函数作为构造函数，通常函数名的首字母大写，此时this指向的是新创建的对象。

tip：使用new操作符，通过造函数创建对象经过了4个过程：1）创建一个新对象；2）将构造函数的作用域赋给新对象【因此this就指向这个新对象】；3）执行构造函数中的代码；4）返回新对象

2.4 apply 和 call

Functioin.prototype.call 和 Function.prototype.apply可以为函数中的this绑定特定的对象。

tip：apply 和 call 的参数传入方式不同。

tip：apply 和 call 是绑定之后执行函数，而bind则是返回绑定之后的函数,返回的函数不能再通过call，apply等改变this的绑定。

2.5 dom事件

结果：dom节点;window;dom节点;window;

MDN对addEventListener的this进行了说明：

通常来说this的值是触发事件的元素的引用，这种特性在多个相似的元素使用同一个通用事件监听器时非常让人满意。

当使用 addEventListener() 为一个元素注册事件的时候，句柄里的 this 值是该元素的引用。其与传递给句柄的 event 参数的 currentTarget 属性的值一样

如果一个事件的属性( 例如. onClick)是指定在一个HTML的元素上的，则这个属性中的JavaScript语句实际上会被包裹在一个处理函数中，在这个处理函数中使用this的效果和使用addEventListener来绑定事件的效果是一样的； this的出现代表了元素的引用。注意到在一个函数里this调用的的效果和标准规则里面是一样的。

click1中this是包裹在处理函数中，此时的this就是处理函数的this，即dom节点。
click2被包裹在处理函数中，这个一个立即执行函数，此时的this相当于简单调用，所以是window。
click3是相当于是对象属性方法的调用，所以this是dom节点。
click4是包裹在处理函数中，相当于函数中简单函数调用，所以此时是window。

2.6 回调函数

回调函数中的this，相当于是简单调用。

3 箭头函数

ES6中新加入了箭头函数，箭头函数中没有this绑定，所以箭头函数中的this是通过作用域链来确定的，且不能通过call，apply或bind改变this的值。

箭头函数中的this是由其外层最近函数（不是箭头函数）中的this决定的。相当于是从作用域链中查找变量this，而作用域链是代码所在的位置决定的。

obj.test()执行是test函数中this指向obj，所以箭头函数中this也就指向来obj。
fn()执行时，test函数中this指向的是window，所以箭头函数中this指向的是window。
